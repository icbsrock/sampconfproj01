Retrospective - Action in Crisis

Action in Crisis is a active reflection of activities/ contingency performed to ensure a commitment/ Timeline activity without impacting any of the application needs while balancing the client emotions and their needs. This is a quick and immediate response to align our focus towards the most imperative MVP's Platform and Pipeline

The below reflections from Team .

1.Platform Demo  - Platform Ready with Spring configurations in place for MVP 1.0 DB Mongo is connected for the back end - The modular components will be wrapped using TF wrapper for those components which are built using CLI

2. Pipeline Demo - First priority is CI to have the working pipeline on XLR with automated gate checks for Git commit hooks → Jenkins (for build) → Sonar (for code quality) → Store image to Artifactory. The second is to integrate the Artifactory to AWS Platform (CD) - Run check readiness 1+ 2 - for 061818

3. Pilot App readiness for On board - 061818 to 062518 - We expect subscription team to provide us a repo to on board one of their services onto CTP Pipeline and platform for service up and running on AWS

4. Ad-hoc Support for Other teams - Please use planning from App teams to consume request for CI/CD on TC platforms. Force intake for CI/CD requests to app teams via PO (Rock)  

5. Working under Pressure - Collaborate as much as possible closely this sprint to make the MVP Happen - Anyone should not wait for anyone else and hold everyone accountable
