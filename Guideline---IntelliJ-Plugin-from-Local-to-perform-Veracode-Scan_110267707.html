CTP : Guideline - IntelliJ Plugin from Local to perform Veracode Scan

Created by Lakshminarayanan Mukundagiri (TMNA), last modified by Unknown User (433661) on May 29, 2018

Installing Veracode Plug-in

  1. Go to https://tools.veracode.com/integrations/IntelliJ/bin/IntelliJ.zip and download the zip file
  2. Select Install plugin from disk....
  3. Navigate to and select the IntelliJ zip file that you downloaded.
  4. Click Apply.
  5. When prompted, restart IntelliJ

To add credentials:

  1. Go to File > Settings > Tools > Veracode > Credentials.
  2. Click Add Credentials.
  3. From the Type dropdown menu, select User/Pass or ID/Key.
  4. In the Identifier column, enter the username and password or your API ID and key.
  5. Click Apply and OK.

 

To Upload Files from IntelliJ:

  1. Go to Veracode > Upload and Scan.
  2. If prompted, enter your Veracode API credentials, either your username/password or ID/key combination. Select the checkbox to store your credentials so that you only have to enter them once.
  3. In the Upload and Scan window, from the Application dropdown menu, select the application for which you want to upload binaries.
  4. After adding the application (if applicable), go to the Upload and Scan window, and in the Scan Type option, select Policy Scan or Sandbox Scan.
    • If you select Sandbox Scan, you must then choose the sandbox from which you want to scan, or follow the prompts to create a sandbox if one does not already exist for this application.
    • If you do not change the scan type, click Create Scan.
  5. In the Create Scan popup, enter the name for the new scan and, optionally, the lifecycle stage.
  6. Click Create.
  7. In the Workspace Files table, use the browse icon to select the files you want to upload from your current projects.
  8. Select Upload.
  9. When prompted to confirm, click Yes to continue the upload.
  10. Click Yes to go directly to the Veracode prescan process after the upload. If you do not want the full scan to continue automatically, click No and, when ready, click Begin Prescan at the top of the Upload Files table.

Using Sandboxes with the IntelliJ Plugin

Within IntelliJ, you create a sandbox into which you upload your files, and then you can run a scan from the sandbox.
To create a sandbox and run a sandbox scan for an application from within IntelliJ:

  1. Select Veracode > Upload and Scan.
  2. If prompted, enter your Veracode API username and password, and select Store username and password so that you only have to enter your credentials one time.
  3. In the Application field, select the application from the dropdown menu and in the Scan Type row, select Sandbox Scan.
  4. Click Create Sandbox and enter a name for the sandbox. If you do not see this button, contact Veracode Support to enable this feature for your account.
  5. In the Workspace Files table, use the browse icon to select the files you want to upload from your current projects. Click Add to select any files not associated with a current project.
  6. Select Upload and, when prompted to confirm, click Yes to continue the upload.
  7. Click Yes to go directly to the Veracode prescan process after the upload. If you do not want the full scan to continue automatically, click No and, when ready, click Begin Prescan at the top of the Upload Files table.
  8. When the prescan is finished, click View Prescan Results at the top of the Upload Files table. 

Working with the Veracode Results

After downloading the Veracode scan results, they appear in the Results view in IntelliJ.

To be able to see Veracode results, you must have the Mitigation API role to be able to mitigate flaws using the Veracode IntelliJ Plugin. 

The Results view lists information about each flaw, including the CWE ID, category, module name, folder path (if available), filename, function name, attack vector, line number, count, severity, exploitability, remediation effort, remediation status and mitigation status. To view additional columns or hide columns, click the icon to the right of the columns.
