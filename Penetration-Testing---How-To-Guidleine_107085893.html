What Is Penetration test for an enterprise & why do we need that ?

Penetration or vulnerability testing software has always been a part of the ethical hacker’s toolkit. More than likely, the customer already is running one or both of these on a regular basis. One of the most exciting developments in pen testing are tools which essentially do all of the hard work from discovery to exploitation, much like an attacker might. There are different forms and method in an organization an information security team may employ to emulate the intended breach behavior for ethical standards to ensure the protection of the systems developed for business value. Penetration testing requires meticulous detail, clarity of the various use cases for the system and thorough skills in order to enter a system from outside the system's network. TMNA Engages our in house Info-sec experts for the needful usually recommended on the Pre Prod environments.


Who within TMNA performs Penetration test ? Contacts and Guidelines?

The Information security team within TMNA is responsible to perform the TMNA Pen test per the request of application team's needs. It is the accountability of every application team to engage and evaluate the need for Pen test with info-sec team via a information security architect's guidance.

Readiness:

  • Stable Code - BB Code Completed and No more version updates to the branch assigned for Pen testing
  • No Major functionality change - During the Pen test
  • No Significant architecture change - During the Pen test
  • Complete Static and dynamic scans using Vera code 
  • Follow ISO, OWASP, NIST guidelines during code development

Environment:

  • Stage (ONLY) - The estimate for the hourly Pen test is roughly around the ball park for > $ 200 / Hr (Need to be equipped to avoid multiple rounds of Pen testing)


Time frame:

  • Request to be made 2 months in advance
  • Testing time depending on application/ testing complexity (4 - 6 Weeks)

Tools:

  • Manual or any appropriate tools


Requirements:

  • Application architecture/ design
  • Functionality overview
  • Account set up/ access

User Account Access

If changes in the team require new user accounts to access Veracode, work with Toyota's Security team to get the necessary accounts created. When requesting a new user account you will need to provide:

  • The name of the Vera-code profile you are requesting access to, note that it is part of  "Project Name"
  • Permissions required - typically this will be permissions to create sandboxes, request scans, and triage/submit mitigation for flaws

 

Testing Profiles

Profiles are split based on which have front-end URLs that can be scanned and which services should be tested through those front-end URLs as part of a team's the development process.If a new service and/or front-end app is being developed, work with Toyota's Security team to determine whether the new code base should belong in one of the existing Vera code profiles or if Security will need to create a new profile.

 

Steps to Scheduling MPT Scans 

 

Be aware: Best practice is to provide 8 weeks notice for scheduling MPT scans, so this process should be started early!

Keep in mind when scheduling the test that it typically takes an additional 3-5 business days after testing completes for the results to be available.

 

  1. Reach out to the Toyota Security team to inform them of the next planned production release date and your corresponding target date for MPT.
  2. Fill out an initial scoping questionnaire spreadsheet. This spreadsheet will be provided by the Toyota Security team who will forward your responses on to the Vera code MPT tester in order to confirm your MPT dates in the schedule.
  3. If needed, the Toyota Security team will help coordinate an additional scoping call between the Veracode MPT tester and the Dev Teams.
  4. Penetration test is required and mandatory for applications at STAGE/ Pre PROD

 

 Guidelines for Scheduling Lead Times 

    • 3 months in advance => no problem scheduling in time
    • 6-8 weeks in advance => most likely will be able to schedule in time
    • 4 weeks in advance => unlikely it will be scheduled in time - the existing MPT schedule would need to be changed to accomodate
    • 2 weeks in advance => there's no way it will happen
