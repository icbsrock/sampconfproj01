Subscription Management Application

Description

Subscription Management application offers various feature for the customers and Dealers acting on behalf of a Customer the following Services;

  • Ability for the Customers to:
    • Buy Products
    • Enroll in Services
  • Upgrade Subscriptions
  • Downgrade Subscriptions
  • Renew Subscriptions
  • Cancel Subscriptions
  • Waive Subscriptions
  • Handle Payments

When the customer makes a subscription, a contract and a SSA is created

High-Level Architecture

The application consists of three layers:

  • UI
  • Middle Tier
  • DB

UI Layer

The UI layer is built using Angular version 5.x and the layer communicates with the middle tier via REST over HTTPS.

Middle Tier

The middle-tier essentially consists of a single micro service (Spring Micro Service) called the Orchestrator with three  CRUD operations:

  • PUT
  • POST
  • DELETE
  • GET

with varying payloads for different services.

Internal Dependencies

The Subscriptions Application will communicate with following internal services once the client signs the contract:

  • Provisioning 
  • CVS
  • Contracts/SSP
  • Contacts 

External Dependencies

The Subscriptions Application will communicate with following internal services once the client signs the contract:

  • Payment System - Please note that the the appropriate Payment system is being Evaluate. Refer to Subscription Management Platform - Comparison

DB Layer

The appropriate DB layer is being evaluated
