SXM Supported API List for CY 17 +


PFB the requested details. @Sajid Kazmi (TMNA), @Roy Lu (TMNA) Vineet Kamdar (TMNA) Ali Kazmi (TMNA) Rock Mutchler (TMS)

 

 

Account Management

Subscription Management

Remote Services

1.      SXM Agents/Agent Portal to consume CTP APIs for customer’s account management, not current D&IS/DCS APIs

2.      List of CTP Account Management APIs (high level):

  • Create Account
  • Search Account
  • Update Address
  • Update Email
  • Update Phone #
  • Delete Account

3.      Phone # will be a mandatory field during account creation

4.      There will not be any use for CustId, GUID will be the only identifier for customers

5.      Notifications for customer’s account management use cases will be sent by CTP

6.      Retire Data Sync – point to point integration

7.      Retire Merge Purge

1.      SXM Agents/Agent Portal to consume CTP APIs for subscription management

2.      List of Subscription Management APIs needed from SXM (high level):

  • Create Subscription
  • Upgrade/Add Product
  • Subscription Cancel
  • Renewal/Update
  • Downgrade/Remove Product
  • Waiver
  • Reversals (RDR)
  • Get Subscription

3.      List of CTP Subscription Management APIs that SXM will consume (high level):

  • Create Subscription
  • Upgrade/Add Product
  • Subscription Cancel
  • Renewal/Update
  • Downgrade/Remove Product
  • Waiver
  • Get Subscription

4.      No Dealer Portal, CTP will build dealer portal

5.      There will not be any use for CustId, GUID will be the only identifier for customers

6.      Notifications for customer’s subscription management use cases will be sent by CTP

7.      NO change to payment processing

8.      NO change to back office integration

9.      NO change to flag management and cust activation

10.   NO change to F&A

11.   Retire Data Sync – point to point integration

12.   Retire Merge Purge

13.   By default, primary subscriber will be assigned as Remote User

14.   No Secondary Subscriber

15.   Ability to process payment in CTP portal

  • SXM to provide API for Cust Activation, Cust Re-Activation
  • TSP service provisioning (CAN/SOS/SVL)

1.      Spec L and Spec I will be triggered by CTP

2.      Remote Auth Code will be generated and verified by CTP

3.      SOS button push, IVR to Invoke CT Remote Auth Request API

4.      Remote User info can be retrieved in Realtime from CT Subscription API

5.      CT SMS Gateway for Shoulder tap for Remote Services
